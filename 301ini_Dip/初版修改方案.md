## `main.c` 定向修改方案：先做最小可运行的 DIP 版

下面这版方案按你的当前思路来：

- **直接在 `main.c` 里改**
- **新增一个 `subevent_result_dip_cb()`**
- **替换旧的 `le_cs_subevent_data_available` 回调**
- **先不删除旧函数，保留参考**
- **先旁路 RAS**
- **先做“直接串口打印本地 IQ”的最小版本**
- **把不再需要的信号量和同步逻辑清掉**

我会尽量写成你后续可以直接丢给 Cursor 的形式。

------

# 一、改造目标

把当前流程从：

```text
subevent_result_cb()
  -> 缓存 latest_local_steps
  -> 等 RAS peer data
  -> ranging_data_get_complete_cb()
  -> cs_de_populate_report()
  -> cs_de_calc()
  -> distance
```

改成：

```text
subevent_result_dip_cb()
  -> 直接解析 result->step_data_buf
  -> 提取本地 PCT
  -> PCT -> local IQ
  -> 串口打印
  -> 等下一次 subevent
```

------

# 二、总体修改策略

## 1. 保留旧代码但不再走旧回调

保留这些内容，先不删：

- `subevent_result_cb()` 旧函数
- `ranging_data_ready_cb()`
- `ranging_data_overwritten_cb()`
- `ranging_data_get_complete_cb()`
- `latest_peer_steps`
- `cs_de_populate_report() / cs_de_calc()` 相关调用代码

但：

- **不再注册旧的 `subevent_result_cb()`**
- **不再订阅 RAS**
- 让系统只走新的 DIP callback

------

## 2. 直接在 `main.c` 里新增 4 个函数

建议新增：

1. `parse_pct_local()`
2. `local_step_iq_cb()`
3. `parse_local_iq_from_subevent()`
4. `subevent_result_dip_cb()`

------

## 3. 清理不再使用的同步逻辑

建议移除或注释掉：

- `sem_local_steps`
- `most_recent_local_ranging_counter`
- `dropped_ranging_counter`

因为新流程里：

- 不再缓存 local step 等 peer
- 不再按 procedure_counter 和 peer 对齐
- 不再需要 local/peer 同步保护

### 2.2 新增 `parse_pct_local()`

```c
static struct bt_le_cs_iq_sample parse_pct_local(const uint8_t pct[3])
{
    uint32_t pct_u32 = pct[0] | (pct[1] << 8) | (pct[2] << 16);

    uint16_t i_u16 = pct_u32 & 0x0FFF;
    uint16_t q_u16 = (pct_u32 >> 12) & 0x0FFF;

    int16_t i = (i_u16 ^ BIT(11)) - BIT(11);
    int16_t q = (q_u16 ^ BIT(11)) - BIT(11);

    return (struct bt_le_cs_iq_sample){
        .i = i,
        .q = q,
    };
}
```

------

### 2.3 新增 step 回调上下文

建议加一个上下文结构，带上 procedure counter：

```c
struct dip_step_parse_ctx {
    uint16_t procedure_counter;
};
```

### 2.4 新增 `local_step_iq_cb()`

这个函数是核心：对每个 step 做解析。

建议第一版实现如下逻辑：

- 只处理 mode 2 / mode 3
- 跳过其他 mode
- 遍历 tone_info
- 取 PCT
- 解析 IQ
- `LOG_INF()` 打印

代码骨架建议如下：

```c
static bool local_step_iq_cb(struct bt_le_cs_subevent_step *step, void *user_data)
{
    struct dip_step_parse_ctx *ctx = user_data;

    if (step == NULL || ctx == NULL) {
        return false;
    }

    if (step->mode == BT_CONN_LE_CS_MAIN_MODE_2) {
        struct bt_hci_le_cs_step_data_mode_2 *data =
            (struct bt_hci_le_cs_step_data_mode_2 *)step->data;

        for (uint8_t tone_index = 0; tone_index < local_n_ap; tone_index++) {
            int antenna_path = tone_index;

            /* 如果 bt_le_cs_get_antenna_path() 可直接调用，可替换为真实映射 */
            /*
            antenna_path = bt_le_cs_get_antenna_path(local_n_ap,
                                                     data->antenna_permutation_index,
                                                     tone_index);
            if (antenna_path < 0) {
                continue;
            }
            */

            if (data->tone_info[antenna_path].quality_indicator !=
                BT_HCI_LE_CS_TONE_QUALITY_HIGH) {
                continue;
            }

            struct bt_le_cs_iq_sample iq =
                parse_pct_local(data->tone_info[antenna_path].phase_correction_term);

            LOG_INF("DIP pc=%u mode=2 ch=%u ap=%d tone=%u I=%d Q=%d",
                    ctx->procedure_counter,
                    step->channel,
                    antenna_path,
                    tone_index,
                    iq.i,
                    iq.q);
        }
    } else if (step->mode == BT_HCI_OP_LE_CS_MAIN_MODE_3) {
        struct bt_hci_le_cs_step_data_mode_3 *data =
            (struct bt_hci_le_cs_step_data_mode_3 *)step->data;

        for (uint8_t tone_index = 0; tone_index < local_n_ap; tone_index++) {
            int antenna_path = tone_index;

            /* 后续可切换到真实 antenna permutation 映射 */
            /*
            antenna_path = bt_le_cs_get_antenna_path(local_n_ap,
                                                     data->antenna_permutation_index,
                                                     tone_index);
            if (antenna_path < 0) {
                continue;
            }
            */

            if (data->tone_info[antenna_path].quality_indicator !=
                BT_HCI_LE_CS_TONE_QUALITY_HIGH) {
                continue;
            }

            struct bt_le_cs_iq_sample iq =
                parse_pct_local(data->tone_info[antenna_path].phase_correction_term);

            LOG_INF("DIP pc=%u mode=3 ch=%u ap=%d tone=%u I=%d Q=%d",
                    ctx->procedure_counter,
                    step->channel,
                    antenna_path,
                    tone_index,
                    iq.i,
                    iq.q);
        }
    }

    return true;
}
```

我这里先用：

```c

```

是为了让第一版尽快跑通。

如果你现在就能直接调用：

```c

```

那你应该换成真实映射。
 如果暂时不方便，就先这样做一个“最小验证版”。

### 2.5 新增 `parse_local_iq_from_subevent()`

这个函数负责复制 `step_data_buf`，然后喂给 `bt_le_cs_step_data_parse()`。

```c
static void parse_local_iq_from_subevent(struct bt_conn_le_cs_subevent_result *result)
{
    if (result == NULL || result->step_data_buf == NULL) {
        return;
    }

    if (result->step_data_buf->len == 0) {
        return;
    }

    uint8_t tmp[LOCAL_PROCEDURE_MEM];
    struct net_buf_simple buf;
    struct dip_step_parse_ctx ctx = {
        .procedure_counter = result->header.procedure_counter,
    };

    if (result->step_data_buf->len > sizeof(tmp)) {
        LOG_ERR("DIP: step_data too large (%u)", result->step_data_buf->len);
        return;
    }

    memcpy(tmp, result->step_data_buf->data, result->step_data_buf->len);
    net_buf_simple_init_with_data(&buf, tmp, result->step_data_buf->len);

    bt_le_cs_step_data_parse(&buf, local_step_iq_cb, &ctx);
}
```

### 2.6 新增新的 callback：`subevent_result_dip_cb()`

```c
static void subevent_result_dip_cb(struct bt_conn *conn,
                                   struct bt_conn_le_cs_subevent_result *result)
{
    ARG_UNUSED(conn);

    if (result == NULL) {
        return;
    }

    if (result->header.subevent_done_status == BT_CONN_LE_CS_SUBEVENT_ABORTED) {
        LOG_WRN("DIP: subevent aborted, pc=%u",
                result->header.procedure_counter);
        return;
    }

    if (result->step_data_buf == NULL || result->step_data_buf->len == 0) {
        LOG_DBG("DIP: no step data");
        return;
    }

    parse_local_iq_from_subevent(result);
}
```

# 六、哪些信号量/变量建议直接清理

如果新 callback 完全替代旧 callback，并且不再走 old local-step cache path，那么以下内容大概率就没用了。

## 1. 删除 `sem_local_steps`

如果有：


建议删掉。

同时删掉所有：


因为现在：

- 不再缓存 local steps 等 peer
- 不再有这个共享资源保护需求

## 2. 删除 procedure-counter 对齐相关变量

如果有：

```c
static int32_t most_recent_local_ranging_counter = PROCEDURE_COUNTER_NONE;
static int32_t dropped_ranging_counter = PROCEDURE_COUNTER_NONE;
```

建议删掉。

因为它们主要是服务于：

- local / peer data 对齐
- dropped data 处理
- RAS-based distance path

新 DIP 不需要。

------

## 3. `latest_local_steps`

### 建议

如果你新 callback 不再缓存本地 step 到 `latest_local_steps`，那么这个 buffer 也可以删。

如果你还担心后面要参考旧代码，那也可以先保留定义，但不再使用。

## 4. `latest_peer_steps`

因为你已经不再走 RAS，所以也可以先不使用。
 是否删除取决于你想不想保留旧链路源码完整性。

## 不再做

# 九、建议的第一版验证方法

改完后重点看这几件事：

## 1. 能否持续收到 subevent callback

你应该能看到 `DIP` 打印持续出现。

## 2. 是否能打印出 channel / mode / I / Q

例如：

```text
DIP pc=10 mode=2 ch=23 ap=0 tone=0 I=123 Q=-42
```

## 3. abort 是否可正常跳过

如果有 aborted subevent，应打印 warning，不应崩。

## 4. 日志量是否太大

如果日志刷太快，先把打印简化成：

- 每个 step 只打印一条
- 或只打印前几个 IQ

否则串口可能成为瓶颈。

## Cursor 任务描述

Modify `main.c` to implement a DIP-only local IQ extraction path.

Requirements:

1. Keep the legacy code in place for reference, but do not use the legacy subevent callback anymore.
2. Add a new callback:
   - `static void subevent_result_dip_cb(struct bt_conn *conn, struct bt_conn_le_cs_subevent_result *result);`
3. Register `subevent_result_dip_cb` as `.le_cs_subevent_data_available`.
4. In `subevent_result_dip_cb`:
   - ignore aborted subevents
   - ignore empty step data
   - directly parse `result->step_data_buf`
   - extract local PCT from mode 2 / mode 3 steps
   - convert PCT to local IQ
   - print local IQ with `LOG_INF`
5. Add helper functions in `main.c`:
   - `parse_pct_local()`
   - `local_step_iq_cb()`
   - `parse_local_iq_from_subevent()`
6. Copy `step_data_buf` into a temporary buffer before parsing, because the parser consumes the buffer.
7. For now, use a fixed `local_n_ap = 1`.
8. Bypass RAS subscription calls in `main()`, but keep the legacy RAS code in the file.
9. Remove no-longer-needed local-step synchronization logic related to waiting for peer data, including:
   - `sem_local_steps`
   - `most_recent_local_ranging_counter`
   - `dropped_ranging_counter`
10. Do not implement distance calculation.
