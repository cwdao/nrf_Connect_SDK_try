# Flash写入模式使用说明

## 概述

现在支持两种flash写入模式：
- **单个写入模式 (SINGLE)**: 每获得一个测距报告立即写入flash
- **批量写入模式 (BATCH)**: 累积多个报告后批量写入flash（原有模式）

## 配置方法

在 `src/flash/flash_ops.h` 中修改 `FLASH_WRITE_MODE` 宏：

```c
// 单个写入模式（测一个写一个）
#define FLASH_WRITE_MODE FLASH_WRITE_MODE_SINGLE

// 批量写入模式（累积后批量写入）
#define FLASH_WRITE_MODE FLASH_WRITE_MODE_BATCH
```

## 两种模式对比

| 特性 | 单个写入模式 | 批量写入模式 |
|------|-------------|-------------|
| **写入时机** | 每个报告立即写入 | 累积27个后批量写入 |
| **内存使用** | 低（无缓冲区） | 高（45个报告缓冲区） |
| **写入速度** | 每次都有flash操作开销 | 减少flash操作次数 |
| **数据安全** | 立即持久化 | 断电可能丢失缓冲区数据 |
| **实时性** | 高（立即写入） | 低（有延迟） |
| **适用场景** | 测试写入性能 | 生产环境长期运行 |

## 新增功能

### 1. Flash状态检查
按键0启动CS前会自动检查flash状态：
- ✅ **Flash干净**: 直接开始
- ⚠️ **使用率<50%**: 提示可继续或擦除
- 🚨 **使用率≥50%**: 强烈建议擦除

### 2. 写入模式显示
启动时会显示当前使用的写入模式。

### 3. 数据编号连续性
无论哪种模式，都确保 `report_index` 连续递增，避免编号跳跃。

## 使用建议

### 测试写入性能时
```c
#define FLASH_WRITE_MODE FLASH_WRITE_MODE_SINGLE
```
- 可以精确测量每个报告的写入时间
- 观察flash写入对测距性能的影响
- 适合短期测试

### 生产环境使用时
```c
#define FLASH_WRITE_MODE FLASH_WRITE_MODE_BATCH
```
- 减少flash写入次数，延长flash寿命
- 更好的系统性能和实时性
- 适合长期连续运行

## 注意事项

1. **修改模式后需要重新编译**
2. **单个写入模式会增加系统负载**
3. **批量模式断电可能丢失部分数据**
4. **两种模式的数据格式完全兼容**

## 保留的原有功能

- 环形缓冲区函数（批量模式仍需要）
- 分片写入功能
- 定时刷新机制
- 按键2读取功能
- 按键3擦除功能

所有原有功能都保持不变，只是增加了新的写入选项。
